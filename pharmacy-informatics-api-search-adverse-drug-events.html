API Search and URL Constructor (Adverse Drug Events)

  • The data on this page is pulled using the adverse drug event API as referenced on the open FDA site https://open.fda.gov/apis/. All disclaimers and limitations from the open FDA site apply to this data retrieval.
  • By default, the API limits results at 100. Limits can be increased to 1000 max. That is limited by the FDA for performance and can impact your data. Limits don't appear to impact received date searches.
  • Each of the searchbar lookups are pulling directly from the API to show exactly how the data is stored. Just start typing and the search starts automatically. When the lookup begins searching, it may take a few seconds longer the first time and faster subsequent times since the browser will cache the results.
    • Also, because of the API result limits, searchbar lookups will only match to the top 1000 items. This can be most notable for generic drug names if there are not a lot of events. Unmatching terms can still be added to the search, and likely makes sense to remove "exact" match in that case.
  • Not to take anything away from the current interactive graphs provided by the FDA. They have a graphical tool too. Both tools have different options and functionality to consider.
    • ***New as of August 2025*** The FDA released a real-time adverse event reporting dashboard. Certainly cool too, but I still like this one's usability to drill down.
  • Use spelling, capitalization, and suffixes of values as they appear exactly in the lookup results below. These values show exactly how openFDA has the data stored, and the URL constructor defaults to matching on the exact text as included.
  • To learn more about how to use this search tool, use the guide here.

Use "include in search" buttons to filter your base query. Any combination of parameters can be used, including multiples of the same field (e.g. Patient Reaction = WHEEZING, ASTHMA, etc)

Use "count by" buttons to group and quantify your query. Only one parameter can be selectable per search and is required to produce results.
